Hello Spring on Kubernetes

This application is using Spring Cloud Kubernetes to connect a frontend app and a backend app.

App container images are built using Cloud Native Buildpacks (look Ma, no Dockerfile!):

$ pack build myrepo/myimage --builder=nebhale/java-build

Try scaling out backend app instances, and see client-side load balancing & service discovery in action:

$ kubectl -n hello-spring-k8s scale --replicas=2 deployment backend

Edit ConfigMap objects and see how the new configuration is applied:

$ kubectl -n hello-spring-k8s edit configmap backend

Hello world!

from backend.hostname
